# 佳明活动导出选项说明

## 概述

现在你可以选择同步不同位置的活动到Strava，而不仅仅是最新的一条。

## 使用方法

### 1. 同步最新活动（默认）
```bash
npm run export_to_strava
```
- 同步第1条活动（最新的）

### 2. 同步倒数第二条活动
```bash
npm run export_second_latest
```
- 同步第2条活动（倒数第二条）

### 3. 按索引同步指定活动
```bash
npm run export_by_index [索引]
```
- `npm run export_by_index 0` - 同步第1条活动（最新）
- `npm run export_by_index 1` - 同步第2条活动（倒数第二条）
- `npm run export_by_index 2` - 同步第3条活动（倒数第三条）
- `npm run export_by_index 5` - 同步第6条活动

## 活动索引说明

| 索引 | 描述 | 活动位置 |
|------|------|----------|
| 0 | 最新活动 | 第1条 |
| 1 | 倒数第二条 | 第2条 |
| 2 | 倒数第三条 | 第3条 |
| 3 | 倒数第四条 | 第4条 |
| ... | ... | ... |

## 示例

### 查看你的活动列表
首先，你可以通过佳明Connect应用或网站查看你的活动列表，了解你想同步哪一条活动。

### 同步特定活动
```bash
# 同步最新的跑步活动
npm run export_to_strava

# 同步昨天的跑步活动（假设是倒数第二条）
npm run export_second_latest

# 同步上周的徒步活动（假设是第5条）
npm run export_by_index 4
```

## 注意事项

1. **活动数量检查**：如果指定的索引超出了你的活动数量，脚本会提示错误
2. **活动类型**：支持所有佳明活动类型（跑步、骑行、徒步等）
3. **文件格式**：自动下载FIT格式文件并上传到Strava
4. **处理时间**：Strava需要几分钟来处理上传的活动

## 故障排除

### 如果提示"没有找到第X条活动记录"
- 检查你的活动数量是否足够
- 确保索引从0开始计算
- 检查佳明账号是否有活动数据

### 如果上传失败
- 检查Strava Access Token是否有效
- 确保网络连接正常
- 检查文件是否下载成功

## 总结

现在你可以灵活地同步任何位置的活动到Strava了！选择合适的方法来同步你需要的活动。
